# Validation

This page documents the numerical accuracy of the NLvib Python port against the
original MATLAB/Octave reference implementation.

## Method

Each comparison notebook in [`notebooks/comparison/`](../notebooks/comparison/) runs the
Python harmonic balance continuation and the MATLAB/Octave reference for the same example,
then overlays both curves on a single figure and computes peak amplitude and peak frequency
errors.

MATLAB data are generated by running the original `.m` scripts via Octave (subprocess call)
and saving results to `.mat` files loaded by `scipy.io.loadmat`.

## Validation Test Suite

Fixture-based regression tests live in `tests/validation/`.  To run them:

```bash
# Generate MATLAB/Octave fixtures (requires Octave on PATH)
python tools/generate_fixtures.py

# Run validation tests
pytest tests/validation/
```

Fixtures are stored in `tests/fixtures/*.npz`.

## Known Accuracy Limits

- **Example 06 (Jenkins element)**: The MATLAB implementation uses a hysteretic
  `elasticDryFriction` model (Jenkins/Masing). This Python port implements the identical
  algorithm in `elastic_dry_friction`. Residual error <5% is dominated by the Galerkin
  truncation error at the harmonic order used, not by an algorithmic difference.

- **Example 08 (beam cubic NMA)**: Three-mode Galerkin reduction produces small amplitude
  discrepancy at high excitation levels where higher beam modes contribute. This matches the
  MATLAB result within 5% and is within the expected accuracy for the chosen truncation.

- **Beam examples (07, 08)**: The finite-difference step `_FD_STEP` used for Jacobian
  estimation must be patched to `1.5e-15` for beam problems where harmonic amplitudes are
  in the `Q ~ 1e-8` regime. See `notebooks/comparison/CONTEXT.md` for details.
